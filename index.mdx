Quarkus is the Kubernetes-native Java framework tailored for OpenJDK HotSpot and GraalVM. It delivers lightning-fast startup times and low memory footprint, making it ideal for cloud-native microservices, serverless functions, and container-based workloads. Quarkus unifies imperative and reactive programming models under one framework, backed by the standards you already know — Jakarta EE, MicroProfile, Hibernate, and Eclipse Vert.x.

## Why Quarkus?

Quarkus shifts as much work as possible to build time — annotations are processed, configuration is validated, and bytecode is generated before your app even starts. This means faster startups, lower memory usage, and a smaller attack surface in production.

## Get Started in 4 Steps

<Steps>
  <Step title="Install the Quarkus CLI">
    Install via SDKMAN!, Homebrew, or download directly:

    ```bash
    sdk install quarkus
    ```
  </Step>
  <Step title="Create a new application">
    Scaffold a new Quarkus project with the REST extension:

    ```bash
    quarkus create app org.acme:my-app --extension='rest' && cd my-app
    ```
  </Step>
  <Step title="Start live coding mode">
    Run in development mode with hot reload enabled:

    ```bash
    quarkus dev
    ```
  </Step>
  <Step title="Call your first endpoint">
    Your app is live at `http://localhost:8080`. Test it with:

    ```bash
    curl http://localhost:8080/hello
    ```
  </Step>
</Steps>
